# 技术说明文档

## 1. 项目概述

### 1.1 项目主要功能
本项目实现了一个面向钓鱼网站检测的多模态深度学习系统，联合利用两类输入信息：

- URL 字符串本身
- 与 URL 对应的网络流量时序数据

系统支持以下完整流程：

- 训练：构建 URL n-gram 词表，加载训练/验证数据，训练多任务模型并保存 checkpoint
- 推理：从 checkpoint 恢复模型、配置和词表，对单条 URL 或批量样本生成预测结果
- 评估：在测试集上输出分类指标、AUC 和阈值等结果
- 数据生成：生成模拟训练数据，便于快速验证流程

### 1.2 项目目标和使用场景
项目目标是对网站样本进行二分类判定，输出其是否为钓鱼站点，并同时给出：

- 钓鱼子类型预测
- 风险分数回归结果

适用场景包括：

- 钓鱼网站离线检测实验
- URL 与流量联合建模研究
- 多任务安全分类模型教学或原型验证

### 1.3 整体运行流程概览
系统整体流程如下：

1. 读取配置参数
2. 从训练集 URL 构建 1/2/3-gram 词表
3. 读取数据文件并将 URL 和流量序列编码为张量
4. 使用 URL 编码器提取 URL 全局表示
5. 使用流量编码器提取流量全局表示
6. 使用跨模态门控模块完成融合
7. 输出二分类 logits、类型分类 logits 和风险分数
8. 在训练阶段计算多任务损失并更新参数
9. 在验证/测试阶段计算指标并输出结果

## 3. 功能结构说明

### 3.1 模块职责划分

#### 配置层
- `PhishingConfig`
  - 集中管理路径、模型超参数、训练超参数、设备参数和输出路径

#### 数据层
- `load_records`
  - 支持读取 JSON 数组和 JSONL 两类格式
- `build_url_vocabs`
  - 从训练集 URL 中构建 1/2/3-gram 词表
- `encode_url_to_ngrams`
  - 将 URL 转换为 n-gram ID 序列
- `parse_traffic_sequence`
  - 将原始流量转为 `[delta_time, size]` 形式
- `PhishingDataset`
  - 负责单样本编码
- `collate_fn`
  - 负责批量拼装、padding 和 mask 生成

#### 模型层
- `models/url_encoder.py`
  - 将 URL 的多粒度 n-gram token 融合后送入 Transformer
- `models/traffic_encoder.py`
  - 将原始流量序列映射到隐藏空间，并通过 Mamba 风格模块建模
- `models/fusion.py`
  - 将 URL 全局向量与流量全局向量进行门控融合
- `models/detector.py`
  - 串联 URL 编码器、流量编码器、融合层和三个任务头

#### 训练与评估层
- `train.py`
  - 负责训练循环、验证、学习率调度与 checkpoint 保存
- `evaluate_test.py`
  - 负责从 checkpoint 恢复模型并在测试集上评估
- `predict.py`
  - 负责推理并输出结构化预测结果

#### 工具层
- `utils.py`
  - 提供通用能力，避免训练、评估、推理脚本重复实现公共流程

### 3.2 模块协作关系

模块之间的协作关系如下：

1. `train.py` 调用 `config.py` 读取配置
2. `train.py` 调用 `dataset.py` 读取训练集和验证集，并构建 URL 词表
3. `train.py` 创建 `models.PhishingDetector`
4. `PhishingDetector` 内部依次调用：
   - `URLTransformerEncoder`
   - `TrafficMambaEncoder`
   - `GateCrossModalFusion`
5. `train.py` 调用 `loss.py` 中的多任务损失函数
6. `train.py` 调用 `utils.evaluate` 完成验证
7. `utils.save_checkpoint` 保存模型权重、配置和词表
8. `predict.py` 与 `evaluate_test.py` 再通过 checkpoint 恢复上述状态

### 3.3 入口文件说明

- 训练入口：`train.py`
- 推理入口：`predict.py`
- 评估入口：`evaluate_test.py`
- 数据生成入口：`data/generate_mock_data.py`

## 4. 算法说明

### 4.1 URL 表征算法

项目不使用手工 URL 特征，而是采用学习型 n-gram 表征：

1. 预处理时仅去除无意义空白字符，保留 URL 原始大小写，不进行统一小写化
2. 分别提取 1-gram、2-gram、3-gram 序列
3. 使用训练集统计频次，构建三套词表
4. 将不同粒度的 token 序列映射为 ID
5. 使用 `NGramEmbedding` 对三组 token 分别嵌入
6. 用门控网络学习 1/2/3-gram 三种粒度在当前位置上的权重
7. 将融合后的 URL token 序列送入 Transformer 编码
8. 取 `CLS` 位置作为 URL 全局表示

之所以在文档中强调保留原始大小写，主要有以下原因：

- URL 中的大小写本身可能携带恶意模式信息，尤其是在路径、参数、token、伪随机串等部分
- 域名部分通常对大小写不敏感，但 URL 的其余部分不适合直接统一转成小写
- 攻击者可能利用大小写混合制造视觉混淆、模式扰动和 token 差异，因此大小写模式本身属于可被模型利用的判别信号

该设计的特点是：

- 保留 URL 字符级和子串级模式
- 用可学习融合代替手工统计特征
- 支持对不同长度 URL 做 padding 和 mask

### 4.2 流量表征算法

流量侧采用最小预处理 + Mamba 风格建模：

1. 输入原始流量记录 `traffic`
2. 将每个元素转换为 `[delta_time, size]`
3. `delta_time` 由当前时间戳与上一个时间戳之差得到
4. 将二维输入通过线性层映射到隐藏空间
5. 在序列前拼接可学习 `CLS` token
6. 叠加位置参数
7. 送入多个 `MambaStyleBlock`

单个 `MambaStyleBlock` 的核心步骤：

1. 先做层归一化
2. 线性层将特征扩展为 `values` 与 `gates`
3. 使用 depthwise 1D 卷积处理时间维局部上下文
4. 使用额外线性层形成状态增强项
5. 使用 `silu(values) * sigmoid(gates)` 实现门控
6. 投影回原始维度并加残差

该设计的特点是：

- 只保留最小必要流量特征
- 对时序局部依赖建模较直接
- 通过门控和残差提高稳定性

### 4.3 跨模态融合算法

`GateCrossModalFusion` 先将 URL 向量和流量向量映射到同一隐藏空间，然后：

1. 将两路特征拼接
2. 通过门控 MLP 生成融合门向量
3. 计算 `gate * url + (1 - gate) * traffic`
4. 计算交互项 `url * traffic`
5. 将融合项和交互项拼接后再投影输出

该机制同时保留：

- 两模态的相对重要性
- 两模态的逐维交互关系

### 4.4 多任务输出与损失

模型包含三个输出头：

- 二分类头：输出是否为钓鱼网站
- 类型分类头：输出钓鱼子类型
- 风险分数头：输出连续风险分值

损失函数组成如下：

- 主任务：`FocalLoss`
- 类型任务：`CrossEntropyLoss`
- 风险任务：`SmoothL1Loss`

总损失计算公式可概括为：

```text
total = lambda_main * main
      + beta_type * type
      + gamma_risk * risk
```

其中主任务使用 Focal Loss，是为了在二分类不平衡场景下抑制简单样本的主导效应。

## 5. 时序流程 / 调用链说明

本节为强制要求部分，以下内容按训练、推理、评估三条主流程分别说明。

### 5.1 训练主流程调用链

#### 执行顺序
1. 进入 `train.py:main`
2. 调用 `get_config`
3. 调用 `setup_logging`
4. 调用 `set_seed`
5. 调用 `get_device`
6. 调用 `load_records` 读取训练集与验证集
7. 调用 `build_url_vocabs` 构建训练词表
8. 调用 `save_url_vocabs` 持久化词表
9. 调用 `build_dataloader` 构建训练/验证 DataLoader
10. 实例化 `PhishingDetector`
11. 调用 `build_criterion`
12. 创建 `AdamW` 和余弦调度器
13. 进入 epoch 循环
14. 每轮调用 `train_one_epoch`
15. 每轮训练后调用 `utils.evaluate`
16. 若验证 AUC 提升，则调用 `save_checkpoint`
17. 若连续若干轮未提升，则触发 early stopping

#### `train_one_epoch` 内部调用链
1. 从 `DataLoader` 取 batch
2. 调用 `move_batch_to_device`
3. 调用 `model(...)`
4. 在 `model(...)` 内部执行：
   - `URLTransformerEncoder.forward`
   - `TrafficMambaEncoder.forward`
   - `GateCrossModalFusion.forward`
   - 三个任务头前向
5. 调用 `criterion(outputs, batch)`
6. 反向传播
7. 梯度裁剪
8. 优化器更新

### 5.2 推理主流程调用链

#### 执行顺序
1. 进入 `predict.py:main`
2. 调用 `parse_args`
3. 调用 `setup_logging`
4. 调用 `get_device`
5. 调用 `load_model_bundle`
6. 在 `load_model_bundle` 中：
   - `load_checkpoint`
   - `PhishingConfig.from_dict`
   - `PhishingDetector(...)`
   - `model.load_state_dict`
7. 根据命令行参数选择单条 URL 或批量文件
8. 调用 `predict_records`
9. 在 `predict_records` 中：
   - `PhishingDataset`
   - `collate_fn`
   - `model(...)`
   - `sigmoid(logits)`
   - `argmax(type_logits)`
10. 输出终端结果或写入 JSON 文件

### 5.3 评估主流程调用链

#### 执行顺序
1. 进入 `evaluate_test.py:main`
2. 调用 `parse_args`
3. 调用 `setup_logging`
4. 调用 `get_device`
5. 调用 `load_checkpoint`
6. 调用 `PhishingConfig.from_dict`
7. 实例化 `PhishingDetector` 并加载权重
8. 调用 `build_criterion`
9. 调用 `load_records`
10. 调用 `build_dataloader`
11. 调用 `utils.evaluate`
12. 在 `evaluate` 中：
    - 遍历 batch
    - `move_batch_to_device`
    - `model(...)`
    - 若提供损失函数则计算 loss
    - 汇总概率与标签
    - 若阈值为空则 `find_optimal_threshold`
    - 计算 accuracy、precision、recall、f1、auc、auc_pr
13. 调用 `save_json` 输出评估结果

### 5.4 模型前向调用关系

`PhishingDetector.forward` 的调用链是整个系统的核心：

1. `URLTransformerEncoder.forward`
   - `NGramEmbedding.forward`
   - `_pad_to_length`
   - `TransformerEncoder`
2. `TrafficMambaEncoder.forward`
   - 多个 `MambaStyleBlock.forward`
3. `GateCrossModalFusion.forward`
4. `binary_head`
5. `type_head`
6. `risk_head`

## 6. 核心类与核心方法说明

### 6.1 `PhishingConfig`

职责：

- 管理训练、评估、推理的统一超参数
- 提供字典序列化与反序列化能力

关键方法：

- `to_dict`
- `from_dict`

### 6.2 `PhishingDataset`

职责：

- 将原始样本记录转换为模型输入
- 统一 URL 编码与流量编码过程

关键输入：

- `url`
- `traffic`
- `label`
- `phish_type`
- `risk_score`

关键输出：

- `ids_1gram`
- `ids_2gram`
- `ids_3gram`
- `traffic`
- 标签字段

### 6.3 `URLTransformerEncoder`

职责：

- 将 URL 从离散 token 序列编码为语义表示

关键输入：

- `ids_1gram`
- `ids_2gram`
- `ids_3gram`
- `url_mask`

关键输出：

- URL 全局向量
- URL 序列表示

### 6.4 `TrafficMambaEncoder`

职责：

- 对 `[delta_time, size]` 时序进行隐藏表示建模

关键输入：

- `traffic_feats`
- `traffic_mask`

关键输出：

- 流量全局向量
- 流量序列表征

### 6.5 `GateCrossModalFusion`

职责：

- 对 URL 与流量两个全局表示进行融合
- 生成统一的下游任务表示

### 6.6 `PhishingDetector`

职责：

- 封装完整多模态多任务模型

关键输出：

- `logits`
- `type_logits`
- `risk_score`
- `pred`

### 6.7 `WeightedMultiTaskLoss`

职责：

- 聚合主任务与两个辅助任务损失

### 6.8 `utils.evaluate`

职责：

- 统一验证集和测试集评估逻辑
- 支持自动阈值搜索

## 7. 补充说明

### 7.1 代码中可见的默认假设

- URL 词表默认只从训练集构建
- 流量输入默认可以解析成 `[timestamp, size]` 或兼容的简化形式
- 二分类阈值默认优先使用 checkpoint 中保存的阈值
- 风险分数最终使用 `sigmoid` 约束到 `[0, 1]`

### 7.2 需要注意的技术风险

- 当前环境若未安装 `torch`，训练、推理与大部分测试无法运行
- 词表大小和最大长度都由配置控制，若样本规模增大需要重新评估显存与训练成本
- 当前流量预处理非常轻量，适合作为最小可运行基线；若实际流量更加复杂，可能需要进一步扩展输入编码策略
- 模拟数据生成脚本仅用于流程验证，不能代表真实安全场景的数据分布

### 7.3 维护建议

- 变更模型结构时，应同步更新：
  - `config.py`
  - `models/`
  - `loss.py`
  - `train.py`
  - `predict.py`
  - `evaluate_test.py`
- 变更数据格式时，应优先检查：
  - `load_records`
  - `parse_traffic_sequence`
  - `PhishingDataset.__getitem__`
  - `collate_fn`
- 若增加新的辅助任务，应同步扩展：
  - `PhishingDetector`
  - `WeightedMultiTaskLoss`
  - `utils.evaluate`

当前测试主要覆盖：

- 数据读取
- 数据集单样本编码
- 批处理拼装
