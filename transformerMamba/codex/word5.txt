# 项目背景
这是一个“基于 URL 字符串与流量时序的多模态钓鱼网站检测系统”。
当前主链路为：

URL n-gram 编码
-> URL Transformer 编码
-> traffic 最小特征解析 [delta_time, size]
-> Mamba 风格流量时序编码
-> 跨模态门控融合
-> 二分类 / 类型分类 / 风险回归多任务输出

当前报告指出的高优先级问题主要有：
1. `dataset.py` 中 `normalize_url` 统一小写，和设计目标不一致，并可能损失大小写模式信息
2. `models/detector.py` 内部 `pred` 固定 0.5，而 `predict.py` / `utils.py` 又有外部 threshold，决策逻辑不统一
3. `predict.py` 中 `predict_records` 一次性拼整批 records，缺少分批推理
4. 数据 schema 校验偏弱，过度依赖默认值
5. `models/fusion.py` 缺少门控行为可观测性
6. `main_logits` 命名语义不清
7. 多任务损失权重目前是静态常数，但这项属于次优先级，不要贸然大改

# 总体目标
请在“不推翻现有主链路、不做无关重构”的前提下，完成一轮 P0/P1 级修正，让代码在以下方面更一致、更可维护、更可解释：

- 数据处理与设计文档一致
- 训练 / 评估 / 推理的决策接口一致
- 推理流程可处理大样本量
- 输入不规范时尽早失败并给出清晰错误
- 关键融合行为具备最小可观测性
- 对外字段命名更清楚，同时尽可能保持兼容

# 执行原则

   - 不要重写多任务头
4. 不要修改训练目标语义，除非是为了修复明显接口不一致。

# 具体任务

## 任务 1：修正 URL 标准化策略（P0）
请重点检查 `dataset.py` 中的 `normalize_url` 及其上下游调用。

目标：
- 去掉“无条件统一小写”的逻辑，默认保留原始大小写模式
- 保留合理的基础清洗，例如去除首尾空白
- 检查这项修改对：
  - n-gram 提取
  - 词表构建
  - 训练/推理一致性
  - checkpoint 恢复
  是否有连锁影响
- 如果你认为直接移除 `.lower()` 会影响旧实验复现，请引入一个显式配置项，例如：
  - `preserve_url_case: bool = True`
  或
  - `lowercase_url: bool = False`
  并让训练 / 推理 / 评估都走同一套配置逻辑

要求：
- 默认行为应与“保留大小写”一致
- 旧行为若保留，必须通过显式配置开启，而不是隐式默认

## 任务 2：统一阈值决策逻辑（P0）
请检查：
- `models/detector.py`
- `predict.py`
- `utils.py`
- 任何涉及 `pred` / `threshold` / `sigmoid(logits)` 的地方

目标：
- 不允许模型内部再硬编码一个固定 0.5 的最终分类决策，除非该值只是临时调试字段且明确标注
- 将“概率 -> 标签”的决策逻辑统一收敛到一个清晰、可复用的位置
- 推荐做法：
  - 模型只输出 logits / probs，不直接决定最终类别
  - 推理与评估统一调用同一个 threshold 决策函数
- 如果需要保留兼容字段，请明确区分：
  - 原始 logit
  - 概率
  - 基于外部 threshold 的 pred

要求：
- 训练、评估、推理三处的决策语义必须一致
- checkpoint 中 threshold 的恢复逻辑要保留
- 不要同时存在“内部 0.5 pred”和“外部 threshold pred”两套真实生效机制

## 任务 3：为推理增加分批处理（P1）
请重点检查 `predict.py` 中的 `predict_records`。

目标：
- 改为按 batch / chunk 分批推理，而不是一次性把所有 records 丢进一次 `collate_fn`
- 支持通过参数控制推理 batch size
- 保证输出顺序和输入顺序一致
- 保证单条样本与多条样本调用语义一致

要求：
- 不要引入复杂依赖
- 尽量复用现有 dataset / collate_fn / dataloader 逻辑
- 对超大输入时内存占用更稳定

## 任务 4：加强输入 schema 校验（P1）
请检查 `dataset.py` 和推理入口处对样本字段的处理。

目标：
- 对关键字段做显式校验，而不是只靠 `dict.get` 默认值兜底
- 区分：
  - 训练必需字段
  - 推理必需字段
  - 可选字段
- 当字段缺失或类型异常时，尽早抛出可读错误信息
- 不要因为默认值把明显脏数据静默吞掉

建议至少检查：
- `url`
- `traffic`
- `label`
- `phish_type`
- `risk_score`

要求：
- 训练和推理的校验规则可以不同，但必须明确
- 错误信息中尽量指出样本索引、字段名、期望类型、实际值摘要

## 任务 5：改善输出命名与兼容性（P1）
请检查 `models/detector.py` 的输出字段。

目标：
- 处理 `main_logits` 命名歧义问题
- 如果它本质上是单个二分类 logit，请改成更清晰的命名，例如：
  - `binary_logit`
  - `binary_logits`（若你确认复数更合适）
- 但为了兼容旧代码，可短期保留别名映射

要求：
- 不要因为重命名导致 train / evaluate / predict 全线断裂
- 若保留兼容字段，请在注释或报告中明确说明该字段未来可废弃

## 任务 6：增加融合层可观测性（P1）
请检查 `models/fusion.py`。

目标：
- 在不改变主计算图语义的前提下，暴露最小必要的融合诊断信息
