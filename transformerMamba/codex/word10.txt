Please inspect and modify the current phishing URL detection project.

The current model has a URL branch and a traffic branch. The traffic branch currently uses raw traffic sequence input, possibly through a Mamba-based sequence encoder.

I want to change the traffic branch input mode so that the model still reads the original raw traffic data from the dataset, but instead of feeding the raw traffic sequence into Mamba, it should extract 30 selected statistical traffic features from each sample's raw traffic sequence and feed those 30 features into a lightweight MLP traffic branch.

Important correction:

Do NOT use traffic_100_features.csv as the direct model input table.
Do NOT align an external feature table to the original dataset by row index.
The 30 features must be computed from each sample's original traffic field inside the data preprocessing / Dataset pipeline.

The raw dataset sample already contains traffic data similar to:

    traffic = [
        [timestamp_1, packet_size_1],
        [timestamp_2, packet_size_2],
        ...
    ]

The new pipeline should be:

    raw traffic sequence
        -> extract selected 30 statistical traffic features
        -> scale features using train-fitted scaler
        -> TrafficFeatureMLP
        -> fusion with URL branch

==================================================

The extractor should support the full set of traffic features that were previously used in the 100-feature analysis, so that any top 30 selected from that analysis can be computed.

At minimum, support these categories:

--------------------------------------------------
6.1 Basic scale and time features
--------------------------------------------------

Examples:

    total_packets
    total_bytes
    packet_rate
    byte_rate

Definitions:

    total_packets = number of packets
    total_bytes = sum(packet_size)

    byte_rate = total_bytes / duration

Use safe division. If duration <= 0, use 0 for rate-like features.

--------------------------------------------------

Divide the traffic duration into 5 equal time bins.

For each bin i = 1..5:

These 10 features describe how packets and bytes are distributed over time.

If duration <= 0, put all packets into bin 1 or return zeros consistently. Add a comment explaining the choice.

Definitions:

    time_to_50pct_bytes:

    time_to_80pct_bytes:
